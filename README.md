# Tone Engine API 🎸

Веб-приложение для гитарного ритейла с уникальной системой умного подбора музыкального оборудования под звучание конкретных музыкальных альбомов. Проект решает проблему поиска нужного звука для начинающих и опытных гитаристов.

---

## 🛠 Стек технологий

*   **Язык:** Java 21
*   **Фреймворки:** Spring Boot 3.5.x (Web, Data JPA, Security)
*   **База данных:** PostgreSQL
*   **Безопасность:** JWT (JSON Web Token), BCrypt
*   **Документация:** OpenAPI 3.0 / Swagger UI
*   **Тестирование:** JUnit 4/5, Mockito
*   **Инструменты:** Maven, Lombok, MapStruct
*   **Контейнеризация:** Docker, Docker Compose

---

## 💡 Функциональность

*   **JWT Аутентификация:** Безопасная регистрация и вход с генерацией access/refresh токенов.
*   **Каталог оборудования:** Просмотр и фильтрация гитар, педалей и усилителей.
*   **Умный Advisor:** Алгоритм скоринга, который анализирует характеристики оборудования и сопоставляет их с пресетами и требованиями выбранного музыкального альбома.
*   **Корзина:** Полноценное управление корзиной (добавление, удаление, очистка) с привязкой к пользователю в БД.
*   **Обработка ошибок:** Глобальный обработчик `@RestControllerAdvice` для стандартизированных JSON-ответов при ошибках.

---

## 🏗 Схема работы приложения

Приложение построено по классической многослойной архитектуре:
1.  **Client:** Отправляет HTTP-запросы с JWT-токеном в заголовке `Authorization: Bearer <token>`.
2.  **Controller Layer:** Принимает запросы, валидирует DTO (Jakarta Validation), вызывает сервисы.
3.  **Security (Filter):** `JwtFilter` перехватывает запросы и аутентифицирует пользователя в `SecurityContextHolder`.
4.  **Service Layer:** Содержит бизнес-логику. `ToneAdvisorService` рассчитывает оптимальный сетап оборудования на основе весов и характеристик.
5.  **Repository Layer:** Spring Data JPA взаимодействует с PostgreSQL. Используются кастомные Query и Specifications для гибкой фильтрации.

## 🚀 Как запустить проект

### Вариант 1: Быстрый запуск через Docker Compose
1. Убедитесь, что у вас установлены Docker и Docker Compose.
2. Клонируйте репозиторий.
3. В корневой директории выполните:
   ```bash
   docker-compose up --build
   ```

### Вариант 2: Запуск вручную
1. Поднимите локальную БД PostgreSQL и создайте базу данных `ToneEngine`.
2. Настройте подключение в `src/main/resources/application.properties`.
3. Соберите проект через Maven:
   ```bash
   mvn clean package
   ```
4. Запустите приложение:
   ```bash
   java -jar target/tone-engine-ver2-2.0.jar
   ```

После запуска Swagger UI будет доступен по адресу: `http://localhost:8080/api/swagger-ui/index.html`

## 📡 Примеры API-запросов

### 1. Регистрация пользователя
**POST** `/api/auth/sign-up`
```json
{
  "name": "john_doe",
  "password": "strongpassword",
  "role": "USER"
}
```

### 2. Авторизация
**POST** `/api/auth/sign-in`
```json
{
  "name": "john_doe",
  "password": "strongpassword"
}
```
*Ответ вернет `accessToken` и `refreshToken`.*

### 3. Подбор оборудования по альбому
**GET** `/api/albums/{id}/products?budget=150000`
*Возвращает список оборудования, наиболее точно подходящего под звучание альбома в рамках указанного бюджета.*
